如何构建 Suspense 数据源

Aug 27, 2024
阅读时长：3 分钟

之前，在没有 Suspense 的时候，是用 ReactLoadable 做异步路由 CodeSplit 并加载 FallbackUI 的；后来，React.lazy 的推出，搭配 Suspense 可以实现同样的功能。

在接触 Next.js 之后，发现服务端异步组件同样可以搭配 Suspense 做代码分割，并实现页面流式渲染。

前段时间，看文档的时候；文档上说【只有启用了 Suspense 的数据源才会激活 Suspense 组件】：

  • 支持 Suspense 的框架如 Relay 和 Next.js；
  • 使用 React.lazy 懒加载组件代码；
  • 使用 use 读取 Promise 的值。

什么是 Suspense 数据源

Next.js 不用多说，ServerComponents 可以直接使用 async 定义异步组件；那么什么是Suspense 数据源呢？我们先看看 React.lazy 和使用 React.use 组件的返回值：

function wrapPromise(promise: Promise<any>) {
  let status = 'pending';
  let resolvedValue: unknown;
  let rejectedError: unknown;
  let suspender = Promise.resolve(promise)
    .then(res => {
      status = 'fulfilled';
      resolvedValue = res;
    })
    .catch(err => {
      status = 'rejected';
      rejectedError = err;
    });
  return {
    getValue() {
      switch (status) {
        case 'pending':
          throw suspender;
        case 'rejected':
          throw rejectedError as any;
        default:
          return resolvedValue as any;
      }
    }
  };
}

export function fetchData() {
  return {
    user: wrapPromise(fetchUser())
  };
}
function fetchUser() {
  return fetch('https://randomuser.me/api')
    .then(x => x.json())
    .then(x => x.results[0]);
}
// 在组件里使用
const resource = fetchData();
export default function User() {
  /**
   * const resource = fetchData();
   * 不可以在组件内部调用
   * 否则，会出错
   */
  const user = resource.user.getValue();
  return (
    <div className="border">
      <p>{user.name.first}</p>
    </div>
  );
}
export default function SuspensePage(props) {
  return (
    <div>
      <Suspense fallback={<div>Loading</div>}>
        <User />
      </Suspense>
    </div>
  );
}
